# מדריך תרומה — SoulDiscipline

תודה על הרצון לתרום! מסמך זה יסייע לך להתחיל מהר, תוך שמירה על איכות עקבית.

## צעדים קצרים
1. **פתח Issue**: תאר את הבעיה/הרעיון.
2. **שוחח עם התחזוקה**: ודא שהכיוון תואם.
3. **צור Fork או Branch**: שם כמו `feature/תיאור-קצר`.
4. **הגדר סביבה**:
   ```bash
   git clone https://github.com/YourUser/discipline.git
   cd discipline
   ```
5. **הרץ בדיקות** לפני שליחת PR.
6. **שלח Pull Request** עם תיאור ברור וקישור ל-issue.

## סטנדרטים
- כתוב קוד נקי, מודולרי ובעל בדיקות.
- שמור על מבנה הקבצים והסגנון (.editorconfig).
- הוסף תיעוד לכל API חדש או שינוי משמעותי.
- קפד על הודעות קומיט קצרות וענייניות.

## בדיקות ותיעוד
- משימות ריצה מומלצות יופיעו תחת `scripts/`.
- בדוק שכל בדיקות `tests/` עוברות לפני פתיחת PR.
- עדכן כל תיעוד רלוונטי ב-`docs/`.

## קוד אתי
כל התרומות נדרשות לעמוד ב־[`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).
