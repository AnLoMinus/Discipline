# אדריכלות המערכת

## מבט-על
המאגר `Discipline` נבנה כבסיס מודולרי לתרגול משמעת עצמית. הוא יוכל לכלול בעתיד שירות Backend, ממשק Frontend וכלי CLI. בשלב זה נשמרת גמישות: את הלוגיקה ממקמים תחת `src/`, ואת הבדיקות תחת `tests/`.

```
Discipline/
├─ src/               # קוד ליבה
├─ tests/             # בדיקות יחידה/אינטגרציה
├─ docs/              # תיעוד רשמי
├─ scripts/           # כלי CLI פנימיים
└─ .github/           # אוטומציה וקהילה
```

## עקרונות תכנון
- **פשטות תחזוקה**: הפרדת שכבות, שמות ברורים, מודולים קטנים.
- **תיעוד מובנה**: כל שינוי משמעותי מלווה ב־ADR או תיעוד ב־`docs/`.
- **בדיקות חובה**: כל מודול חדש מצריך בדיקות.
- **גמישות טכנולוגית**: ניתן לחבר בקלות סטאק Python או Node.js.

## מפת דרך טכנית (טיוטה)
1. CLI ראשוני לבניית רוטינות יומיות (`src/cli`).
2. API REST או GraphQL לנתוני משתמש.
3. ממשק UI (React/Svelte) בנתיב `src/ui`.
4. הטמעת אחסון נתונים (SQLite/PostgreSQL).

עדכונים נוספים יתועדו ב־[`docs/adr`](adr/README.md).
