# מדריך סגנון

## קוד
- השתמשו ב־PEP8 עבור Python, וב־ESLint + Prettier עבור JavaScript/TypeScript.
- שמרו על שמות קבצים באנגלית, באותיות קטנות עם מקפים (`kebab-case`) לסקריפטים, ו־`snake_case` למודולים בפייתון.
- פונקציות קצרות, מודולריות ומבדיקות היטב.

## תיעוד
- כתיבה בעברית רהוטה, קונסיסטנטיות במונחים.
- כותרות בפורמט Title Case או פסוקים רלוונטיים.
- היעזרו ב־Markdown סטנדרטי, הוסיפו טבלאות ורשימות כשצריך.

## הודעות קומיט
- `[קטגוריה] תיאור קצר` כגון `feat: מוסיף CLI בסיסי`.
- עשו שימוש ב־English לתיאור הפעולה.

## תוויות Issues/PR
- `type:feature`, `type:bug`, `type:docs`, `type:chore`.
- `status:in-progress`, `status:needs-review`, `status:blocked`.
