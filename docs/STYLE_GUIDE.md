# מדריך סגנון

## תיעוד
- כתיבה בעברית רהוטה, קונסיסטנטיות במונחים.
- כותרות בפורמט Title Case או פסוקים רלוונטיים.
- היעזרו ב־Markdown סטנדרטי, הוסיפו טבלאות ורשימות כשצריך.
- הדגישו מסרים מרכזיים באמצעות **מודגש** ואייקונים רלוונטיים בלבד.
- שמרו על קישורים מעודכנים (GitHub, אתר חי, מסמכים פנימיים).

## גרסת HTML סטטית
- הקפידו ששינויים ב־Markdown ישתקפו גם בגרסאות ה־HTML.
- הימנעו מסקריפטים כבדים; העיצוב נשאר ב־`styles.css` בלבד.
- שמרו על תאימות RTL מלאה.

## הודעות קומיט
- שימוש באותיות לטיניות: `docs:`, `chore:`, `site:` וכדומה.
- תיאור קצר וממוקד שמבהיר איזה מסמך או חלק באתר השתנה.
